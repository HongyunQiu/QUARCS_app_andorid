# 安卓轻量页面自动化测试方案

目标：在不手工点手机屏幕的前提下，实现：

1. 一键构建与部署 APK
2. 自动启动 App
3. 自动打开轻量调试页
4. 自动填写串口/波特率
5. 自动点击 `Connect`、`Self Test`、`Move`、`Stop`
6. 自动抓取页面日志

## 方案核心

不要走 `adb shell input tap` 这种黑盒坐标点击。

更稳的方案是：

1. 用 [deploy_android.sh](/home/q/workspace/QUARCS_app_andorid/deploy_android.sh) 完成构建、安装、启动
2. 用 `adb forward` 把手机内的本地服务映射到电脑：
   - `http://127.0.0.1:18080/`
   - `ws://127.0.0.1:8600/`
3. 在电脑上直接访问同一套轻量调试页面
4. 用 Playwright 自动操作页面元素

这有几个好处：

- 不依赖手机分辨率、坐标、输入法
- 不依赖 UI 文案截图识别
- 和手机里 App 使用的是同一套后端桥接服务
- 失败时可直接抓页面日志做诊断

## 已提供脚本

脚本路径：
[test_android_lite_ui.sh](/home/q/workspace/QUARCS_app_andorid/test_android_lite_ui.sh)

默认流程：

1. `QUARCS_EMBEDDED_UI_MODE=lite` 构建并部署
2. 启动 `org.qtproject.example.QUARCS_app`
3. 转发 `18080/8600`
4. 用 Playwright 打开轻量页面
5. 自动执行 focuser 冒烟流程
6. 保存页面日志到 `doc/android_lite_ui_test_*.log`

脚本默认使用 Playwright 默认浏览器。
如果你显式指定 `BROWSER_CHANNEL=chrome`，但本机没有系统 Chrome，脚本会自动回退到默认浏览器。

## 运行方式

最常用：

```bash
cd /home/q/workspace/QUARCS_app_andorid
./test_android_lite_ui.sh
```

指定设备：

```bash
DEVICE=10AF952KJG002PX ./test_android_lite_ui.sh
```

指定串口和步数：

```bash
SERIAL_PORT=/dev/ttyACM0 BAUD_RATE=9600 MOVE_STEPS=200 ./test_android_lite_ui.sh
```

只复用当前已安装 App，不重新部署：

```bash
DEPLOY_FIRST=0 ./test_android_lite_ui.sh
```

可视化浏览器执行：

```bash
HEADED=1 ./test_android_lite_ui.sh
```

首次安装 Playwright 浏览器：

```bash
INSTALL_BROWSER=1 ./test_android_lite_ui.sh
```

指定浏览器通道：

```bash
BROWSER_CHANNEL=chrome ./test_android_lite_ui.sh
```

## 当前自动化覆盖

当前脚本默认覆盖 focuser 最小烟雾流程：

1. 等待页面连接桥接服务
2. 填写串口
3. 保存波特率
4. 点击 `FocuserConnect`
5. 点击 `FocuserSelfTest`
6. 点击一次 `Move Left`
7. 点击 `Stop`
8. 导出页面日志

## 后续建议

这套自动化建议分两层维护：

1. 页面层烟雾测试
   - 验证页面能打开
   - 按钮能点
   - 返回日志能显示

2. 桥接层回归测试
   - 直接对 `ws://127.0.0.1:8600/` 发命令
   - 校验返回值和状态机
   - 适合做更稳定的 CI 回归

页面层更接近真实用户操作。
桥接层更稳定、执行更快。
推荐两层并行保留。
