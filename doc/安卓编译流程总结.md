# QUARCS_app 安卓编译流程总结（Qt 6.5.3）

> 适用目录：`~/workspace/QUARCS_app`  
> 目标产物：`android-build-debug.apk`

## 1. 环境与前提

- 系统：Ubuntu 24.04（ubuntu2404 节点）
- Qt：6.5.3（含 `android_arm64_v8a`）
- JDK：建议 17
- Android SDK/NDK：已安装并可被 Qt Creator/构建脚本识别
- ADB：可连接测试手机（USB 或无线）
- 网络：建议配置代理（下载 Gradle/依赖更稳定）

## 2. 关键准备检查

在开始构建前，建议先确认：

1. Qt Android 模块是否齐全（至少包含）：
   - `QtWebView`
   - `QtBluetooth`
2. 资源文件大小写是否一致（Linux 区分大小写）：
   - 例如 `qml.qrc` 中引用 `img/Btn_set.png`，但实际文件是 `img/btn_set.png`，会直接导致构建失败。
3. 当前工作目录是否正确：
   - `~/workspace/QUARCS_app`

## 3. 标准构建流程（Qt 路线）

### 步骤 A：进入工程目录

```bash
cd ~/workspace/QUARCS_app
```

### 步骤 B：配置并生成 Android 构建目录

> 说明：实际可通过 Qt Creator 或命令行触发 qmake/cmake；本项目历史记录中产物目录是：

- `build-qt6-android-arm64/android-build/`

### 步骤 C：执行 Gradle 打包

```bash
cd ~/workspace/QUARCS_app/build-qt6-android-arm64/android-build
./gradlew assembleDebug --info
```

### 步骤 D：确认 APK 产物

```bash
ls -lh ~/workspace/QUARCS_app/build-qt6-android-arm64/android-build/build/outputs/apk/debug/
```

预期文件：

- `android-build-debug.apk`

## 4. 安装与启动验证

```bash
adb devices
adb install -r ~/workspace/QUARCS_app/build-qt6-android-arm64/android-build/build/outputs/apk/debug/android-build-debug.apk
```

## 5. 常见问题与处理经验

- Gradle 下载卡住：优先走代理并预热缓存。
- `qml.qrc` 资源找不到：重点检查文件名大小写。
- `android_openssl openssl.pri not found`：当前链路中为 warning，不阻塞构建。

## 6. 推荐迭代节奏

每次改动执行闭环：Build -> Install -> Launch -> Log。
